# PARK Agent - サービス仕様書

## 1. サービス概要

- **一言で**: 営業特化の日本語チャット1画面。議事録が裏に溜まり、聞けば答え、頼めば作る
- **対象ユーザー**: 営業担当（デモ・プロトタイプ検証用）
- **解決する課題**: SFA/CRM/ダッシュボードを飛ばし「話しかけるだけで営業管理が成立する」体験の検証
- **ステータス**: プロトタイプ（新規1画面）

## 2. 画面一覧

| 画面     | パス | 概要 |
|----------|------|------|
| 営業AIチャット | `/` | ヘッダー・左サイド（お気に入り・最近の議事録）・チャットエリア（ウェルカム・サジェスト・履歴・入力欄） |

## 3. 技術スタック

- フロント: Vite, React, TypeScript, Tailwind（Navy #1e3a5f / Orange #e85d04）
- バックエンド: Express, Gemini API
- データ: 議事録モック（サーバー内）

## 4. 非機能要件・検証

### プロトタイプ評価（UI/UX アンケート）

- **目的**: 営業PMなど最大約1万人を想定した利用者に対し、プロトタイプの「使いやすさ」「実際に動いたら使えるか」を検証し、**UI/UX の厳格な査定**を行う。
- **成果物**: 設問一覧・実施手順・集計の重点項目は [docs/SURVEY_UX.md](docs/SURVEY_UX.md) に記載する。
- **実施**: Google フォーム等で実施。プロトタイプ利用開始時または初回タスク完了後に画面内でアンケートリンクを表示することを推奨。
- **集計**: 「やや悪い」「とても悪い」の割合・NPS・「そのまま本番配布してよいか」の分布・自由記述のテーマ分類を重点的に確認し、改善優先度をつける。
