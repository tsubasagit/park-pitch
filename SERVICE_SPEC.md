# Pitch - サービス仕様書

## 1. サービス概要

- **一言で**: 自社サービス特化の提案書AIジェネレーター。PDF→構造化→提案書自動生成
- **対象ユーザー**: プロフェッショナルファーム（士業・コンサル）の営業担当
- **解決する課題**: 複数サービスを組み合わせた提案書作成の手間を自動化。自社の専門家体制・サービス情報を蓄積し、高品質な提案書をAIで即時生成
- **ステータス**: MVP
- **MVPターゲット**: アクシスグループ（axis-gp.com）— 徳島拠点の士業系ファーム、116名・4拠点・8法人

## 2. ユーザーロールと権限

| ロール | できること |
|---|---|
| 管理者（営業担当） | 会社プロフィール設定、サービス登録・編集・削除、提案書生成・閲覧・印刷 |

## 3. 機能一覧

### 実装済み
- [x] 会社プロフィール設定（初回オンボーディング付き）
- [x] サービス登録（PDF→AI構造化抽出→編集→保存）
- [x] 提案書生成（複数サービス選択＋ヒアリング3項目→AI生成）
- [x] 提案書プレビュー・印刷（Markdownページ分割表示＋@media print）
- [x] 提案書履歴一覧

### 未実装（予定）

- [ ] チーム共有・権限管理

## 4. 画面一覧

| 画面 | パス | 概要 |
|---|---|---|
| 会社プロフィール | `/settings` | 初回オンボーディング＋会社情報編集（理念・専門家体制・拠点・強み） |
| サービス管理 | `/services` | サービス一覧＋PDF登録→構造化抽出→編集 |
| 提案書作成 | `/generate` | 提案先情報＋複数サービス選択＋ヒアリング3項目→生成 |
| 提案書履歴 | `/proposals` | 過去の提案書一覧 |
| 提案書プレビュー | `/proposals/:id` | Markdown表示＋印刷 |

## 5. データモデル

### JSON永続化（server/data/）
- `company.json` — 会社プロフィール（1件）
  - 主要フィールド: name, industry, philosophy, expertise, offices, strengths[], contactPerson, email
- `services.json` — サービス一覧（配列）
  - 主要フィールド: id, name, category, overview, targetClients, challengesSolved[], deliverables[], expertType, engagementType, pdfFilename
- `proposals.json` — 提案書一覧（配列）
  - 主要フィールド: id, serviceIds[], serviceNames[], clientInfo, hearingInput, markdownContent, createdAt

## 6. 外部連携

- AI: Gemini API（PDF構造化抽出＋提案書生成）
- ホスティング: ローカル（Express + Vite dev server）

## 7. ビジネスルール

- PDF抽出失敗時は空フィールドで返却（ユーザーが手動入力可能）
- 提案書生成にはサービス1つ以上の選択が必須
- ヒアリング3項目（要望・課題、予算感、導入時期）は全て必須
- 会社プロフィール未設定時はオンボーディング画面にリダイレクト

## 8. 非機能要件

- PDFアップロードサイズ: 10MB制限
- 提案書生成: 15-30秒（Gemini API依存）
- 印刷: A4対応、@media printでページ分割

## 9. 差別化（堀）

- **堀A: スマート商品登録** — PDF→構造化データ抽出。ユーザーが磨くほど提案書品質が上がる蓄積資産
- **堀B: 会社プロフィール** — 専門家体制・拠点・強みが全提案書に自動反映。ブランド化された出力
