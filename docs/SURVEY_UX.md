# PARK Agent 営業PM向け UI/UX アンケート調査

## 概要

- **目的**: プロトタイプの「使いやすさ」「実際に動いたら使えるか」の検証と、**UI/UX の厳格な査定**
- **対象**: 営業PMなど、プロトタイプ利用者（想定最大約1万人）
- **所要時間目安**: 3〜5 分
- **前提**: 本ツールはチャットでサマリーを取り出すことが目的。会社一覧・案件・議事録は Google Drive に格納し、一覧閲覧は外部リンクで提供する想定。

## スタンドアロン版（テスト・紙用ひな形）

[docs/survey_form_standalone.html](survey_form_standalone.html) に、上記設問をそのまま反映した HTML フォームを用意しています。ブラウザで開いてテストや紙のアンケート原稿として利用できます。本番で回答を収集する場合は、Google フォームを作成しそのURLを配布してください。

---

## Google フォームで作成する場合の手順

1. 新しいフォームを作成し、タイトルを「PARK Agent プロトタイプ UI/UX アンケート」などに設定する。
2. セクションごとに「セクション」を追加し、上記の「セクション1〜6」の見出しを付ける。
3. 各設問を上記の「推奨タイプ」に合わせて追加する。
   - ラジオ: 単一選択の「ラジオボタン」
   - チェックボックス: 「チェックボックス」
   - スケール: 「リニアスケール」（最小・最大を指定）
   - 段落: 「段落」テキスト
4. セクション3・4の「共通の選択肢」は、各質問の選択肢として同じ6つを設定する。
5. セクション5の前に、説明文を「説明テキスト」で追加することを推奨する。
6. 回答は匿名で収集する場合は「メールアドレスを収集しない」にし、必要に応じて「任意でメールを記入」の設問を追加する。

## 集計時の重点確認項目

- **UI/UX 厳格査定**: セクション3の各項目について「やや悪い」「とても悪い」の割合を算出し、改善優先度をつける。
- **NPS**: セクション2-3の 0–10 を集計し、推奨者(9–10)・受動的(7–8)・批判的(0–6)の分布を確認する。
- **本番配布可否**: セクション2-4（そのまま本番配布してよいか）の分布を確認する。
- **「直すべき」複数選択**: セクション3-16の選択率を集計し、特に多い項目を優先して改善する。
- **自由記述**: テーマごとに分類し、厳しい指摘・要望を一覧化する。

---

## 実施方法の目安

- **配布**: プロトタイプ利用開始時または初回タスク完了後に、画面内バナー／モーダルでアンケートリンクを表示すると回収率が高い。必要に応じてメール・Slack でも配布。
- **匿名**: 匿名にすると厳しい意見が集まりやすい。フォローアップ希望者のみ任意でメールを記入してもらう形式も可。
- **インセンティブ**: 可能であれば抽選で謝礼や景品を用意すると、完了率・自由記述の質が上がりやすい。
